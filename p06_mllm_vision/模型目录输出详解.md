# 模型目录输出详解

## 概述

路径：`outputs/exp3_full_no_lora/final/`

该目录是 **Qwen2.5-VL**（视觉语言模型）经过全参数微调（Full Fine-tuning，无 LoRA）后保存的最终模型输出目录。目录总大小约 **15 GB**，包含模型权重、配置文件、分词器文件等完整推理所需的全部文件。

模型基本信息：
- **架构**: `Qwen2_5_VLForConditionalGeneration`（Qwen2.5 视觉语言条件生成模型）
- **参数规模**: ~15GB（float32），约 **3.75B 参数**（15,018,491,904 bytes / 4 bytes per float32）
- **语言模型层数**: 36 层 Transformer
- **视觉编码器层数**: 32 层 ViT blocks
- **词表大小**: 151,936

## 各文件详细解析

### 1. `config.json` — 模型架构配置

这是模型最核心的配置文件，定义了模型的完整架构。`from_pretrained()` 加载模型时首先读取此文件。

#### 顶层字段

| 字段 | 值 | 说明 |
|------|-----|------|
| `architectures` | `["Qwen2_5_VLForConditionalGeneration"]` | 模型类名，决定加载哪个模型实现 |
| `model_type` | `"qwen2_5_vl"` | 模型类型标识 |
| `hidden_size` | `2048` | 语言模型隐藏层维度 |
| `intermediate_size` | `11008` | FFN 中间层维度（≈5.4x hidden_size） |
| `num_hidden_layers` | `36` | Transformer 层数 |
| `num_attention_heads` | `16` | 注意力头数 |
| `num_key_value_heads` | `2` | GQA（分组查询注意力）的 KV 头数 |
| `hidden_act` | `"silu"` | 激活函数（SiLU/Swish） |
| `max_position_embeddings` | `128000` | 最大序列长度 |
| `vocab_size` | `151936` | 词表大小 |
| `rms_norm_eps` | `1e-06` | RMSNorm 的 epsilon |
| `rope_theta` | `1000000.0` | RoPE 位置编码基础频率 |
| `initializer_range` | `0.02` | 权重初始化标准差 |
| `tie_word_embeddings` | `true` | 输入和输出 embedding 共享 |
| `torch_dtype` | `"float32"` | 模型存储精度 |
| `transformers_version` | `"4.51.3"` | 使用的 transformers 版本 |
| `use_cache` | `true` | 推理时使用 KV cache |
| `attention_dropout` | `0.0` | 注意力 dropout 比率 |

#### `rope_scaling` 字段 — RoPE 缩放配置

```json
{
  "mrope_section": [16, 24, 24],
  "rope_type": "default",
  "type": "default"
}
```

| 子字段 | 说明 |
|--------|------|
| `mrope_section` | M-RoPE（多维 RoPE）各维度分配的频率通道数：时间维 16，高度维 24，宽度维 24（共 64 = num_heads × head_dim / 2 中的一部分） |
| `rope_type` | RoPE 类型，default 表示标准旋转位置编码 |

#### Sliding Window Attention 字段

| 字段 | 值 | 说明 |
|------|-----|------|
| `sliding_window` | `32768` | 滑动窗口大小 |
| `max_window_layers` | `70` | 使用滑动窗口的最大层数（70 > 36，即全部层都不使用滑动窗口） |
| `use_sliding_window` | `false` | 当前未启用滑动窗口 |

#### 视觉相关 Token ID

| 字段 | 值 | 说明 |
|------|-----|------|
| `image_token_id` | `151655` | 图像占位 token（`<\|image_pad\|>`） |
| `video_token_id` | `151656` | 视频占位 token（`<\|video_pad\|>`） |
| `vision_start_token_id` | `151652` | 视觉内容起始 token |
| `vision_end_token_id` | `151653` | 视觉内容结束 token |
| `vision_token_id` | `151654` | 通用视觉占位 token |
| `bos_token_id` | `151643` | 序列起始 token |
| `eos_token_id` | `151645` | 序列结束 token |

#### `vision_config` 字段 — 视觉编码器配置

```json
{
  "depth": 32,
  "hidden_size": 1280,
  "num_heads": 16,
  "intermediate_size": 3420,
  "hidden_act": "silu",
  "in_channels": 3,
  "patch_size": 14,
  "spatial_patch_size": 14,
  "temporal_patch_size": 2,
  "spatial_merge_size": 2,
  "window_size": 112,
  "out_hidden_size": 2048,
  "fullatt_block_indexes": [7, 15, 23, 31],
  "tokens_per_second": 2,
  "torch_dtype": "float32"
}
```

| 子字段 | 说明 |
|--------|------|
| `depth` | ViT 层数 = 32 |
| `hidden_size` | ViT 隐藏维度 = 1280 |
| `num_heads` | ViT 注意力头数 = 16 |
| `intermediate_size` | ViT FFN 中间维度 = 3420 |
| `in_channels` / `in_chans` | 输入通道数 = 3（RGB） |
| `patch_size` / `spatial_patch_size` | 空间 patch 大小 = 14×14 像素 |
| `temporal_patch_size` | 时间 patch 大小 = 2（视频每 2 帧合并） |
| `spatial_merge_size` | 空间合并倍数 = 2（相邻 2×2 个 patch 合并为 1 个 token） |
| `window_size` | 视觉注意力窗口大小 = 112（局部注意力窗口） |
| `out_hidden_size` | 输出投影维度 = 2048（与语言模型 hidden_size 对齐） |
| `fullatt_block_indexes` | 使用全局注意力的层索引（每 8 层一次） |
| `tokens_per_second` | 视频中每秒采样的 token 数 |

### 2. `generation_config.json` — 推理生成配置

控制模型在推理（`model.generate()`）时的行为。

```json
{
  "attn_implementation": "eager",
  "bos_token_id": 151643,
  "do_sample": true,
  "eos_token_id": [151645, 151643],
  "pad_token_id": 151643,
  "repetition_penalty": 1.05,
  "temperature": 1e-06,
  "transformers_version": "4.51.3"
}
```

| 字段 | 说明 |
|------|------|
| `attn_implementation` | 注意力实现方式：`"eager"` 表示标准 PyTorch 实现（非 FlashAttention） |
| `do_sample` | 启用采样（配合 temperature 使用） |
| `eos_token_id` | 结束 token 列表：`<\|im_end\|>` (151645) 和 `<\|endoftext\|>` (151643) |
| `pad_token_id` | 填充 token = `<\|endoftext\|>` |
| `repetition_penalty` | 重复惩罚系数 1.05（轻微抑制重复） |
| `temperature` | 采样温度 = 1e-06（近似贪心解码，但仍走采样路径） |

### 3. `model-0000X-of-00004.safetensors` — 模型权重文件

模型权重以 **safetensors** 格式存储（比 pickle 更安全、加载更快），分为 4 个分片：

| 分片 | 大小 | 包含内容 |
|------|------|----------|
| `model-00001-of-00004.safetensors` | 4.7 GB | 视觉编码器全部 32 层（`visual.blocks.*`）+ 视觉 merger + `model.embed_tokens` + 语言模型 layer 0~3 部分权重 |
| `model-00002-of-00004.safetensors` | 4.6 GB | 语言模型 layer 3~19 部分权重 |
| `model-00003-of-00004.safetensors` | 4.6 GB | 语言模型 layer 19~35 部分权重 |
| `model-00004-of-00004.safetensors` | 173 MB | 语言模型 layer 35 的 MLP + `model.norm.weight`（最终 RMSNorm） |

#### 每层包含的权重张量

**语言模型每层**（`model.layers.{i}`）：
- `input_layernorm.weight` — 输入层归一化权重
- `self_attn.q_proj.weight` / `.bias` — Query 投影
- `self_attn.k_proj.weight` / `.bias` — Key 投影
- `self_attn.v_proj.weight` / `.bias` — Value 投影
- `self_attn.o_proj.weight` — 输出投影
- `post_attention_layernorm.weight` — 注意力后归一化权重
- `mlp.gate_proj.weight` — SwiGLU 门控投影
- `mlp.up_proj.weight` — SwiGLU 上投影
- `mlp.down_proj.weight` — SwiGLU 下投影

**视觉编码器每层**（`visual.blocks.{i}`）：
- `attn.qkv.weight` / `.bias` — QKV 合并投影
- `attn.proj.weight` / `.bias` — 注意力输出投影
- `norm1.weight` — 第一层归一化
- `norm2.weight` — 第二层归一化
- `mlp.gate_proj.weight` / `.bias` — 门控投影
- `mlp.up_proj.weight` / `.bias` — 上投影
- `mlp.down_proj.weight` / `.bias` — 下投影

**视觉 Merger**（`visual.merger`）：
- `ln_q.weight` — Query 层归一化
- `mlp.0.weight` / `.bias` — 第一个线性层（1280×4 → 2048）
- `mlp.2.weight` / `.bias` — 第二个线性层

**其他**：
- `model.embed_tokens.weight` — token embedding 矩阵（151936 × 2048）
- `model.norm.weight` — 最终 RMSNorm
- `visual.patch_embed.proj.weight` — 视觉 patch embedding 卷积核

控制分词器行为和聊天模板。

| 字段 | 值 | 说明 |

| `add_bos_token` | `false` | 不自动添加 BOS |
| `add_prefix_space` | `false` | 不自动在输入前加空格 |

#### Chat Template 解析

```
{% for message in messages %}
  <|im_start|>{{ role }}
  {{ content }}<|im_end|>
{% endfor %}
```

模板支持多模态消息格式：
- 图像内容 → `<|vision_start|><|image_pad|><|vision_end|>`
- 视频内容 → `<|vision_start|><|video_pad|><|vision_end|>`
- 文本内容 → 直接输出
- 可选：`add_vision_id` 标记自动添加 "Picture N:" / "Video N:" 前缀

### 8. `merges.txt` — BPE 合并规则

文本文件（1.6 MB），每行一条 BPE 合并规则：

```
#version: 0.2
Ġ Ġ
ĠĠ ĠĠ
i n
Ġ t
ĠĠĠĠ ĠĠĠĠ
e r
...
```

- 第一行：版本标识
- 后续每行：两个 token 的合并规则（按优先级排序）
- `Ġ` 代表空格（byte 0x20 的编码）
- `Ċ` 代表换行符（byte 0x0A 的编码）

### 10. `special_tokens_map.json` — 特殊 Token 角色映射

```json
{
  "additional_special_tokens": [
    "<|im_start|>", "<|im_end|>",
    "<|object_ref_start|>", "<|object_ref_end|>",
    "<|box_start|>", "<|box_end|>",
    "<|quad_start|>", "<|quad_end|>",
    "<|vision_start|>", "<|vision_end|>",
    "<|vision_pad|>", "<|image_pad|>", "<|video_pad|>"
  ],
  "eos_token": "<|im_end|>",
  "pad_token": "<|endoftext|>"
}
```

**作用**：定义各特殊 token 在分词器中的语义角色，确保这些 token 不会被拆分。每个 token 的属性：
- `lstrip` / `rstrip`：是否去除左右空格
- `normalized`：是否参与归一化
- `single_word`：是否仅匹配完整单词

### 11. `training_args.bin` — 训练参数

Python pickle 序列化的 `TrainingArguments` 对象（5.3 KB），保存训练时的全部超参数配置，包括：

- 学习率、batch size、梯度累积步数
- 优化器类型、权重衰减、warmup 策略
- 训练 epoch 数、保存策略
- 混合精度设置（bf16/fp16）
- 分布式训练配置
- 输出目录等

> 注：此文件为二进制格式，需用 `torch.load()` 或 `pickle.load()` 读取。

---

### 12. safetensors 格式说明

**safetensors** 是 Hugging Face 推出的安全张量存储格式，相比传统 pickle：

| 特性 | safetensors | pickle (`.bin`) |
|------|-------------|-----------------|
| 安全性 | 无代码执行风险 | 可执行任意代码 |
| 加载速度 | 零拷贝/mmap 支持 | 需完整反序列化 |
| 延迟加载 | 支持（按需加载单个张量） | 不支持 |
| 文件头 | JSON 元数据，可快速解析 | 无 |

## 模型加载方式

```python
from transformers import Qwen2_5_VLForConditionalGeneration, AutoProcessor

model = Qwen2_5_VLForConditionalGeneration.from_pretrained(
    "outputs/exp3_full_no_lora/final",
    torch_dtype="auto",
    device_map="auto"
)
processor = AutoProcessor.from_pretrained("outputs/exp3_full_no_lora/final")
```

加载流程：
1. 读取 `config.json` → 实例化模型架构
2. 读取 `model.safetensors.index.json` → 定位各张量所在分片
3. 按需加载 `model-0000X-of-00004.safetensors` → 填充模型参数
4. 读取 `tokenizer_config.json` + `tokenizer.json` → 初始化分词器
5. 推理时参考 `generation_config.json` 设置生成参数
