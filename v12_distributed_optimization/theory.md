# 分布式训练与工程优化原理

## 1. 数据并行 vs 模型并行

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| 数据并行 (DP/DDP) | 每张卡完整模型，数据切分 | 模型能放下单卡 |
| 模型并行 (MP) | 模型按层切分到多卡 | 超大模型 |
| 流水线并行 (PP) | 模型按阶段切分，微批次流水 | 超大模型 + 低通信 |
| 张量并行 (TP) | 单层内部切分 (如 MLP 切列) | 需要高速 NVLink |

## 2. DeepSpeed ZeRO

ZeRO (Zero Redundancy Optimizer) 消除数据并行中的冗余：

| Stage | 切分内容 | 显存节省 |
|-------|---------|---------|
| Stage 1 | 优化器状态 | ~4x |
| Stage 2 | + 梯度 | ~8x |
| Stage 3 | + 模型参数 | ~N倍(N=GPU数) |

### ZeRO Stage 2 (最常用)
- 优化器状态和梯度在 GPU 间分片
- 前向/反向时每卡仍有完整参数 → 计算无变化
- 更新时 AllReduce 梯度后各卡只更新自己分片的参数

## 3. PyTorch FSDP

Fully Sharded Data Parallel：PyTorch 原生的类 ZeRO-3 实现。
- 参数、梯度、优化器状态全部分片
- 按需 All-Gather 参数用于计算
- 配合 activation checkpointing 可训练超大模型

## 4. 混合精度训练

| 精度 | 位数 | 范围 | 用途 |
|------|------|------|------|
| FP32 | 32-bit | 大 | 主权重备份 |
| FP16 | 16-bit | 小(易溢出) | 前向/反向计算 |
| BF16 | 16-bit | 大(精度低) | A100+ 推荐 |

GradScaler：FP16 训练时自动缩放 Loss，防止梯度下溢。

## 5. 推理优化

- **量化 (INT8/INT4)**：模型大小减半到四分之一，速度提升 2-4x
- **ONNX 导出**：跨框架部署，配合 TensorRT 可极大加速
- **Flash Attention**：O(n) 显存的高效注意力实现
- **vLLM / TGI**：专业的 LLM 推理框架

## 6. 关键优化技巧总结

| 技巧 | 作用 | 实现难度 |
|------|------|---------|
| 梯度累积 | 小显存模拟大 batch | 简单 |
| 梯度检查点 | 时间换空间 | 简单 |
| 混合精度 | 速度 2x，显存 0.5x | 简单 |
| DeepSpeed ZeRO-2 | 显存 ~8x | 中等 |
| Flash Attention | 速度/显存大幅优化 | 安装即用 |
| 量化推理 | 推理速度/成本 | 简单 |
